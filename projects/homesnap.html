Homesnap

Agent Client Connection

Brand/Project:

CoStar Group, Homesnap, Residential Real Estate

Role:

UX Designer

Date:

Q1 2022

Context:

The Homesnap team had recently released the “Showings” feature which assisted listing (seller) real estate agents in creating and coordinating showings on the platform. The team needed further functionality to add additional listing agents, buyer agents, and owner/occupants to an existing showing.

The Problem

How do we make a seamless experience for listing agents to add other agents and home owners to their showing?

* Additional Considerations:

  • This functionality needs to be applied across the brand’s desktop web, mobile web, iOS, and Android platforms

The Solution

First, list out all contact types. Then, meet with developers and PMs from the web and native apps to understand how the interaction fits into each of their patterns for adding contact.

Users

Agent: Alex Gonzales

  • Acts mostly on the seller side
  • Needs better way to coordinate showings with owners and other agents
  • Active Homesnap user
  • Busy schedule with many active clients

Main Required Actions

The user needs to be able to add contacts to their showing, these can be other agents or owner/occupants of the home they’re showing. Whether a contact is connected (known or unknown) with them on Homesnap is also important.
Contact types:
  1. Agent, known
  2. Agent, unknown
  3. Owner/occupant, known
  4. Owner/occupant, unknown

Development Considerations

Homesnap existed on web and native platforms, each with their own pattern for interacting with and adding contacts. The interaction flow for each would need to fit the existing software design for each platform.

Actions Flows

  • Web pattern begins with checking if a contact is known
  • Native pattern begins with checking the user type

The Results

After some additional cycles of feedback with developers I created high fidelity prototypes for all platform types.
